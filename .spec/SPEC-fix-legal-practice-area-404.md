## Description

Clicking any legal practice area link in the footer (e.g., "Corporate Law", "Personal Injury") shows "Page Not Found" instead of the hub page. The pages exist and the routing dispatches correctly, but the component reads the wrong URL parameter name.

Root cause: `LegalPracticeAreaPage` calls `useParams<{ practiceArea: string }>()` and looks up `params.practiceArea`. But the route that renders it is `/legal/:slug` (via `LegalSlugRouter`), so the param is named `slug` — not `practiceArea`. The lookup returns `undefined`, `meta` is `undefined`, and the component renders its "Page Not Found" fallback.

## Codebase Analysis

The routing uses a slug-based dispatcher: `/legal/:slug` is caught by `LegalSlugRouter` which checks the slug against known Sets and renders the appropriate component. `LegalCityPage` correctly reads `useParams<{ slug }>`. `LegalPracticeAreaPage` was written expecting a `:practiceArea` param (as if it had its own dedicated route) but is actually rendered via the shared `:slug` route - so it must read `params.slug`.
